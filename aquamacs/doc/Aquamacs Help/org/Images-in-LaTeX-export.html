12.6.5 Images in LaTeX export

Images that are linked to without a description part in the link, like ‘[[file:img.jpg]]’ or ‘[[./img.jpg]]’ will be inserted into the PDF output file resulting from LaTeX processing. Org will use an \includegraphics macro to insert the image. If you have specified a caption and/or a label as described in Images and tables, the figure will be wrapped into a figure environment and thus become a floating element. You can use an #+ATTR_LaTeX: line to specify the various options that can be used in the optional argument of the \includegraphics macro. To modify the placement option of the figure environment, add something like ‘placement=[h!]’ to the Attributes.

If you would like to let text flow around the image, add the word ‘wrap’ to the #+ATTR_LaTeX: line, which will make the figure occupy the left half of the page. To fine-tune, the placement field will be the set of additional arguments needed by the wrapfigure environment. Note that if you change the size of the image, you need to use compatible settings for \includegraphics and wrapfigure.

     #+CAPTION:    The black-body emission of the disk around HR 4049
     #+LABEL:      fig:SED-HR4049
     #+ATTR_LaTeX: width=5cm,angle=90
     [[./img/sed-hr4049.pdf]]

     #+ATTR_LaTeX: width=0.38\textwidth wrap placement={r}{0.4\textwidth}
     [[./img/hst.png]]

If you need references to a label created in this way, write ‘\ref{fig:SED-HR4049}’ just like in LaTeX.
